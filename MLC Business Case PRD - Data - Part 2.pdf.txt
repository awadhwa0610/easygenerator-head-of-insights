Companies building onboarding or training materials for global teams require 
courses to be translated into multiple languages. While it ºs possible to translate 
a course into multiple languages on EG, here are the key problems with the 
current solution:
Editing and managing existing translated courses: when changes need to

translated. The higher the number of courses and translations, the more 
time-consuming and difficult it gets.

Assumptions, Outstanding Issues, and Key 
Decisions
QÓÇí What other solution approaches did we explore?
Answer: Before implementing the current MLS, we explored a different 
approach. We looked at ‚ÄúBundling Courses into a SCORM PackageÀÆ where 
the author selects multiple languages and bundles them into one SCORM 
package. We discarded this approach because it was a hack, it would make 
future development difficult, and it didn ºt solve 2 core use cases:
Editing and managing existing translated courses
Tracking course results.
QÓÇí What ºs the maximum size of the SCORM package we will support? What ºs 
the max. of languages we will support?
Answer: There are two limitations: 1ÓÇÇ set by AWS translations service and 2ÓÇÇ 
SCORM size limit of the LMS. For AWS, the downloading is capped at 15min 
and different LMSes will have different limits (e.g. ÓÇ£100MB to ÓÇ£500MBÓÇÇ. 
We ºve decided that we won ºt set any limit until we see critical feedback 
after beta milestone 1 release. If we see that the SCORM downloads fail 
frequently, we will communicate in-product to make customers aware. We 
won ºt be able to affect the size limit on the LMSes side. The customer will 
have to set the size limit accordingly in EG before uploading it to their LMS.
